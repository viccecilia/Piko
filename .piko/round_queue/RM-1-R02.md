# Round ID: RM-1-R02

Round Name: Opt-In And Rate-Limit Policy

本轮目标:

Make the real-market discovery opt-in and bounded before any connector is allowed to run.

本轮任务:
- 执行任务:
  - Ensure real-market collection requires explicit flags such as `PIKO_ENABLE_DISCOVERY_REAL_SOURCE=true` and `PIKO_LIVE_DISCOVERY_TEST=true`.
  - Add or document source endpoint env vars for Steam, Reddit, JP, KR, and SERP/search.
  - Add limits for max sources, max records per source, timeout, and user agent.
  - Add policy text explaining that default test and client paths remain fixture/offline.
- 测试任务:
  - Test real-market collection is disabled by default.
  - Test missing endpoint configuration fails clearly.
  - Test limit values are bounded and cannot request unbounded collection.
- 协作验收任务:
  - Worker summary must list every env flag and every configured endpoint name.

允许修改:

- `packages/shared/config.py`
- `packages/discovery/*`
- `docs/*`
- `tests/test_discovery_search.py`
- `.piko/summaries/worker_RM-1-R02.md`
- `.piko/round_status.json`

禁止修改:

- Do not add broad crawler behavior.
- Do not enable real collection by default.
- Do not call real external services in default tests.

必须运行的验证:

- `python -m pytest tests\test_discovery_search.py -q`
- `python -m pytest`

完成定义:

- Real-market collection is impossible without explicit opt-in and endpoint configuration.
- Default Piko client and tests remain offline/fixture-safe.
