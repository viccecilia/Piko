# Round ID: V02-4-R02

Round Name: Run Trace And Observability Artifact

本轮目标:

建立本地 run trace artifact，让 operator 能看到每次 workflow 的阶段、Gate、失败原因和验证命令。

本轮任务:
- 执行任务:
  - 定义 `artifacts/v02_runtime/latest_run_trace.json`。
  - 字段至少包含 run_id、stages、agents、tools、gates、failures、commands、artifact_refs、status。
  - 不记录 secrets、raw source body 或长文。
- 测试任务:
  - JSON parse probe。
  - trace 不含 secrets/raw_text/authorization。
  - failed gate 可被定位。
- 协作验收任务:
  - 生成 `.piko/summaries/worker_V02-4-R02.md`。

允许修改:

- `packages/*`
- `tests/*`
- `docs/*`
- `artifacts/v02_runtime/*`
- `.piko/summaries/worker_V02-4-R02.md`
- `.piko/round_status.json`

禁止修改:

- 不要接入外部 observability SaaS。
- 不要保存敏感信息。

必须运行的验证:

- Run trace artifact tests。

完成定义:

- Piko 有本地可复现的 run trace 结构。

输出格式:
- 修改了什么
- 每个任务状态
- 验证结果
- 协作验收结果
- 未完成/风险
- 下一轮建议
