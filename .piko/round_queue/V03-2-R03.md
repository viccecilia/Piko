# Round ID: V03-2-R03

Round Name: Retry Failure And Gate Semantics

本轮目标:

补齐 practical workflow 必须具备的 retry、failure、Gate semantics，避免 adapter 只是形似。

本轮任务:
- 执行任务:
  - 为 local graph runner 添加受控 retry/failure 模拟。
  - 定义哪些失败可 retry，哪些必须 blocked_for_operator。
  - Gate 输出必须包含 decision、reason、required_evidence、next_action。
- 测试任务:
  - 测试 retry count、failure trace、Gate blocked 行为。
  - 测试失败不会被标记为 verified/pass。
- 协作验收任务:
  - 生成 `.piko/summaries/worker_V03-2-R03.md` 和 `.piko/summaries/worker_V03-2.md`。

允许修改:

- `packages/*`
- `tests/*`
- `artifacts/v03_practical_plugin/*`
- `.piko/summaries/worker_V03-2-R03.md`
- `.piko/summaries/worker_V03-2.md`
- `.piko/round_status.json`

禁止修改:

- 不放宽 Gate。
- 不把 failed trace 写成 passed。

必须运行的验证:

- Retry/failure/Gate tests

完成定义:

- Adapter pilot 具备实战需要的失败处理语义。
- Operator 能从 trace 看到失败原因和下一步。
