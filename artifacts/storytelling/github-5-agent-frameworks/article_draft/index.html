GitHub Agent Radar / 小红书图文版

我模拟跑了一遍 GitHub 上 5 个高星 Agent 项目，发现普通人先看这 5 个就够了

如果你最近开始研究 AI Agent，大概率会遇到一个问题：项目太多了。LangGraph、CrewAI、AutoGen、LlamaIndex、Dify、OpenAI Agents SDK……每一个看起来都重要，但真正适合普通人先搞懂的，是它们分别代表的 5 个系统位置。

这次我没有只看 README，也没有只按 star 数排序，而是做了一个模拟任务：

同一个任务，分别用 5 类 Agent 框架跑一遍。任务很简单：让 Agent 找资料、整理证据、分工写草稿、互相讨论，最后把能力做成一个能用的入口。

跑完之后我发现，普通人没必要一上来学几十个 Agent 项目。先看懂这 5 个代表就够了：LangGraph、LlamaIndex、CrewAI、AutoGen / AG2、Dify / OpenAI Agents SDK。

为什么不是随便挑 5 个？

因为一个真正能工作的 Agent 系统，至少要回答 6 个问题：

  • 任务怎么拆？
  • 状态怎么保存？
  • 工具怎么调用？
  • 证据从哪里来？
  • 多个 Agent 怎么协作？
  • 最后结果怎么验证？

很多人卡住，不是因为模型不够强，而是因为只学了其中一块。只学工具调用，很快会发现流程乱；只学 RAG，很快会发现不会安排任务；只学多 Agent，很快会发现每个 Agent 都在聊天，但结果没人收口。

第 1 个：LangGraph，负责流程和状态

LangGraph流程 / 状态 / Gate

模拟体验：最像“Agent 的流程编排器”。它不负责让模型更聪明，而是让任务知道下一步该去哪、状态放在哪里、失败后怎么回到某一步。

为什么要选它：因为只要 Agent 任务超过两步，你就会需要流程和状态。LangGraph 代表的是“把 Agent 跑稳”的能力。

第 2 个：LlamaIndex，负责资料和证据

LlamaIndex资料 / 索引 / 追溯

模拟体验：最像“Agent 的资料室”。它擅长把文档、网页、知识库变成可检索、可引用、可追溯的上下文。

为什么要选它：因为 Agent 不能只会说话，还要知道自己根据什么说。LlamaIndex 代表的是“让回答有证据”的能力。

第 3 个：CrewAI，负责角色和协作

CrewAI角色 / 任务 / 分工

模拟体验：最像“把一个复杂任务拆成一组岗位”。你可以让 Researcher 找资料，让 Writer 写草稿，让 Reviewer 挑问题。

为什么要选它：因为普通人最容易理解“角色分工”。CrewAI 代表的是“把 Agent 协作讲清楚”的能力。

第 4 个：AutoGen / AG2，负责多 Agent 实验

AutoGen / AG2对话 / 协商 / 实验

模拟体验：最像“多 Agent 实验室”。你可以观察多个 Agent 怎么对话、协商、互相调用工具，以及什么时候开始跑偏。

为什么要选它：因为它能让你看到多 Agent 的真实边界：有时很有启发，有时只是热闹。AutoGen / AG2 代表的是“做协作实验”的能力。

第 5 个：Dify / OpenAI Agents SDK，负责应用和工具入口

Dify / Agents SDK工具 / 模型 / 应用

模拟体验：最像“把 Agent 做成产品入口”。Dify 更偏低代码应用搭建，OpenAI Agents SDK 更偏开发者把工具、模型、handoff 组合起来。

为什么要选它：因为最后你总要让用户真的能用。它们代表的是“把 Agent 从 demo 变成应用”的能力。

为什么这 5 个就够入门？

因为它们刚好覆盖一个 Agent 系统从内部执行到外部应用的完整链路：

  • LangGraph：任务怎么跑。
  • LlamaIndex：资料怎么来。
  • CrewAI：角色怎么分。
  • AutoGen / AG2：多个 Agent 怎么实验。
  • Dify / OpenAI Agents SDK：怎么做成入口。

如果你是普通人，学习顺序怎么排？

我的建议不是从最火的开始，而是按“系统位置”学：

先学这三个
  • LangGraph：看懂流程
  • LlamaIndex：看懂证据
  • Dify / Agents SDK：看懂应用入口
再看这两个
  • CrewAI：看懂角色分工
  • AutoGen / AG2：看懂多 Agent 实验
  • 不要一开始就追所有框架

看任何新 Agent 项目，都问这 6 个问题

以后你再刷到一个新的 Agent 项目，不要先问“它是不是最强”，先问：

  • 这个项目在我的系统里负责哪个位置？
  • 它替代了什么旧流程？
  • 它能不能被测试？
  • 它失败了能不能回滚？
  • 它有没有证据链？
  • 它会不会绕过人工确认？

如果回答不了这些问题，再高星也只是收藏夹里的一个链接。

一句话总结

这 5 个 Agent/框架不是唯一答案，但它们是最适合入门的一张地图。

如果你想搞懂 AI Agent，不要一上来收藏 100 个项目。先搞懂 5 个位置：流程怎么跑，证据怎么来，角色怎么分，协作怎么实验，应用怎么落地。
