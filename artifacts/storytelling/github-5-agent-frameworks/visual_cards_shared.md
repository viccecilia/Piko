# 图文/短视频共用视觉卡片方案

主题：我让 Codex 每天扫描 GitHub 高星 AI 项目，最后发现 Agent 真正该抄的不是框架，而是这 5 个能力

## 视觉定位

- 画幅：9:16，优先适配小红书图文、视频封面、短视频关键帧。
- 风格：深色技术知识卡，像一张正在被整理的 Agent 能力地图。
- 核心画面语言：GitHub repo 卡片、能力地图、流程节点、证据链、角色协作、多 Agent 对话、工具入口、验证 Gate。
- 不使用外部图片；所有画面用 HTML/CSS 图形、文字卡、流程线和图标化符号表达。

## 统一设计系统

- 背景：`#090d12`，带细网格、扫描线、轻微噪点感。
- 主文字：`#f4f7fb`。
- 次级文字：`#a9b4c2`。
- 卡片底色：`#121923`。
- 结构线：`#263241`。
- 五类能力色：
  - 流程 LangGraph：`#5aa9ff`
  - 证据 LlamaIndex：`#47d18c`
  - 分工 CrewAI：`#f0b84a`
  - 实验 AutoGen/AG2：`#b48cff`
  - 应用 Dify/Agents SDK：`#ff6b6b`
- 字体：系统中文黑体优先，标题 700-900，正文 400-600。

## 卡片结构

每张卡固定 4 层：

1. 顶部元信息：`GitHub Agent Map / 01-12`
2. 主标题：不超过 18 个中文字符，视频可直接做大字幕。
3. 视觉主体：能力图、流程线、repo 卡、角色卡或 Gate。
4. 底部判断：一句话结论，适合做视频口播落点。

## 优化后的 12 张共用卡

### 01 封面

主标题：我让 Codex 每天扫描 GitHub 高星 AI 项目

副标题：真正该抄的不是框架，而是 5 个能力

画面：一堆 repo 卡从四周收束成 5 张发光能力卡。

用途：视频封面、小红书 P1、公众号首图。

### 02 痛点

主标题：项目越收藏，越乱

底部判断：问题不是项目少，是不知道它们各自负责什么。

画面：LangGraph、CrewAI、AutoGen、LlamaIndex、Dify 等 repo 卡片堆叠，路线线条打结。

### 03 反转

主标题：不是学 100 个项目

副标题：而是筛出 5 个系统位置

画面：漏斗从“高星项目”筛到“流程 / 证据 / 分工 / 实验 / 应用”。

### 04 总地图

主标题：Agent 的 5 个位置

画面：五列能力地图。

底部判断：先看位置，再看项目名。

### 05 LangGraph

主标题：流程和状态

关键问题：下一步谁做？状态放哪里？失败怎么重试？

画面：Node -> Checkpoint -> Gate -> Retry -> Done。

### 06 LlamaIndex

主标题：资料和证据

关键问题：资料从哪里来？回答能不能追溯？

画面：Source -> Index -> Evidence -> Answer。

### 07 CrewAI

主标题：角色和协作

关键问题：谁找资料？谁写草稿？谁审核？

画面：Source、Evidence、Writer、Editor、Factcheck 角色卡围绕任务协作。

### 08 AutoGen / AG2

主标题：多 Agent 实验

关键问题：多个 Agent 怎么对话、协商、互相调用？

画面：三组 Agent 对话气泡，最后收敛到 Result。

### 09 Dify / OpenAI Agents SDK

主标题：应用和工具入口

关键问题：怎么接工具、接模型、做成用户可用入口？

画面：Tool、Model、File、API 汇入 App UI。

### 10 错误问题

主标题：别问哪个最强

副标题：问它负责哪个位置

画面：左边“排名焦虑”被划掉，右边“系统位置”被点亮。

### 11 Piko 路线

主标题：成熟框架 + Piko 规则

副标题：证据链 + 验证 + 人类确认

画面：Framework -> Piko Rules -> Evidence -> Verify -> Human Gate。

### 12 结尾

主标题：先看系统位置

副标题：再看项目名字

底部判断：真正能落地的 Agent，是一条能被验证的生产线。

画面：五张能力卡锁进一张架构板。

## 视频复用规则

- 每张卡都可以作为一个短视频 scene 的视觉主体。
- 视频主字幕使用卡片主标题，不把整段口播塞进画面。
- 关键帧节奏：封面 4 秒，痛点 6 秒，反转 6 秒，五类能力各 6-8 秒，Piko 路线 8 秒，结尾 6 秒。
- 所有英文项目名保留原文，中文解释放在下一层。
- 不放星数，避免发布前数据过期；如需星数，发布前再查 GitHub 当前值。
