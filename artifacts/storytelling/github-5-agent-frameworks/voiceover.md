# Voiceover Script

大多数人学 Agent，第一步就错了。

他们一上来就收藏项目。

LangGraph、CrewAI、AutoGen、LlamaIndex、Dify。

越看越多，越学越乱。

但我看完一圈 GitHub 高星 Agent 项目之后，反而觉得：

普通人不用掌握那么多。

真正该先搞懂的，是 5 类能力。

第一类，流程。

代表是 LangGraph。

它解决的是，任务下一步谁做，状态放哪里，失败怎么重试。

第二类，证据。

代表是 LlamaIndex。

它解决的是，资料从哪里来，回答能不能追溯。

第三类，分工。

代表是 CrewAI。

它把复杂工作拆成多个角色，比如找资料、写草稿、做审核。

第四类，实验。

代表是 AutoGen 或 AG2。

它适合观察多个 Agent 怎么对话、协商、互相调用。

第五类，应用。

代表是 Dify 或 OpenAI Agents SDK。

它让 Agent 能接工具、接模型，变成用户真的能用的入口。

所以别问哪个 Agent 最强。

要问它在系统里负责哪个位置。

流程、证据、分工、实验、应用。

这 5 个位置看懂了，你再看任何 Agent 项目，都不会乱。

这也是我做 Piko 的路线。

不训练模型。

不从零造 Agent runtime。

用成熟框架。

把业务规则、证据链、验证体系喂进去。

最后由人类确认。

真正能落地的 Agent，不是一个工具。

是一条能被验证的生产线。
