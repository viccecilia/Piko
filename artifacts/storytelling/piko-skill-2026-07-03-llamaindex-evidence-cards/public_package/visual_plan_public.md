# 视觉卡片与视频设计说明

## 总体风格

- 平台：小红书图文、抖音竖屏视频、公众号配图。
- 比例：9:16 竖屏，卡片建议 1080x1440 或 1080x1920，视频 1080x1920。
- 风格：知识类、技术感、节奏快、大字幕、卡片和流程图为主。
- 颜色：深墨色文字、白色底、青色强调、少量绿色表示“可追溯”、少量红色表示“风险”。
- 字体：系统无衬线，标题粗体，正文短句，不做长段落。

## 共用视觉模块

1. GitHub 项目卡
   - 项目名：LlamaIndex
   - Stars：约 50.6k，发布前复核
   - Keywords：Document AI / RAG / Index / Retrieval
   - 注意：只模拟公开项目介绍卡，不伪装成真实网页截图。

2. 资料卡
   - 字段：来源、时间、主题、适用范围、可信度。
   - 用于解释“元信息”。

3. 来源卡
   - 三张卡片：Doc A、Doc B、Note C。
   - 用于解释“回答带来源”。

4. 五步地图
   - 资料进入系统 -> 加元信息 -> 建索引 -> 检索相关片段 -> 回答带来源。
   - 小红书 P9 和视频结尾共用。

## 小红书图文结构

- P1: 强钩子封面。
- P2: 常见错误。
- P3: 核心判断。
- P4: 项目定位。
- P5: 关键能力。
- P6: 普通人可复用的方法。
- P7: 适合场景。
- P8: 不适合场景。
- P9: 收藏型学习地图。

## 视频结构

- 0-6s: 痛点钩子，快速抓注意力。
- 6-14s: 项目露出，说明今天讲 LlamaIndex。
- 14-24s: 反常识判断，不只是 RAG。
- 24-34s: 解释“依据在哪”的痛点。
- 34-58s: 讲机制，资料整理、元信息、索引和检索。
- 58-68s: 五步学习地图。
- 68-76s: 适合场景。
- 76-80s: 总结定格。

## 不使用的素材

- 不使用未经许可的外部图片。
- 不使用真人出镜。
- 不使用具体真人声音克隆。
- 不展示任何敏感凭据。
- 不展示本地路径或私人文件名。
