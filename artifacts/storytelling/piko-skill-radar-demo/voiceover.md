# 口播稿

我最近给 Piko 加了一个新想法。

叫每日 Skill 雷达。

它解决的不是“找工具”。

而是判断一个工具，到底值不值得进入系统。

因为 AI 项目很容易遇到这个问题。

今天你接了一个 Agent 框架。

明天 GitHub 上又出现一个更强的。

那到底换不换？

不能凭感觉。

Piko 要每天扫描高星开源项目、Agent 框架、RAG 工具、插件系统和自动化工具。

然后给每个能力打分。

看它准不准，稳不稳，贵不贵，安不安全，license 有没有风险。

但注意，它不会自动改代码。

它只会输出升级提案。

真正进入系统，还要经过 Worker 实现，Verify 验证，最后人类确认。

这件事对 Piko 很重要。

因为 Piko 不是只做游戏攻略。

它要变成一个可插拔的多 Agent 系统。

今天能做游戏玩家痛点。

以后也能做电商差评、SaaS 反馈、旅游攻略、本地生活。

所以它必须知道：

哪些能力该保留。

哪些能力该替换。

哪些能力风险太高，不能碰。

我的目标是：

让系统自己发现任务，自己选择工具，自己执行，自己验证。

人类只做最后一步确认。

这才是一个长期 AI 系统真正该有的样子。
