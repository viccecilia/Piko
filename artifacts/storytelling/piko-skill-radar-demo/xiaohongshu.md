# 小红书图文卡片稿

## P1 封面

我给 Piko 加了一个  
**每日 Skill 雷达**

让 AI 系统每天自己找更好的能力

## P2 旧痛点

AI 项目最麻烦的不是接工具。

而是：

今天这个 Agent 很强，  
明天又出现一个更好的。

到底换不换？

## P3 新能力

Skill 雷达每天会看：

- GitHub 高星项目
- Agent 框架
- RAG 工具
- 插件系统
- 自动化工具
- 验证/Guardrail 项目

## P4 它不自动改代码

这点很重要。

它不会看到新工具就乱接。

它只做三件事：

1. 发现
2. 评分
3. 提案

## P5 判断标准

一个 Skill 值不值得进 Piko，要看：

- 准确性
- 稳定性
- 成本
- 安全
- license
- 可维护性
- 是否适合当前任务

## P6 升级流程

扫描日报  
-> 升级提案  
-> Worker 实现  
-> Verify 验证  
-> 人类确认

## P7 最终目标

Piko 以后应该能自己：

- 找任务
- 选工具
- 执行
- 验证
- 生成结果

人类只做最后确认。

## P8 一句话总结

真正长期有用的 AI 系统，  
不是会用很多工具。

而是知道什么时候该换工具。

## P9 结尾

我会继续把 Piko 做成  
一个可插拔、可验证、会自我升级的多 Agent 系统。
